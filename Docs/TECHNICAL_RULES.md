# TECHNICAL_RULES

## Regras de Plataforma
- Plataforma inicial oficial: **PC Windows**.
- Android é apenas possibilidade futura, sem prioridade atual.
- Priorizar entrada por **teclado** na fase inicial.
- Estruturar suporte a **controle** para futuras etapas.

## Regras de Escopo e Integração
- **Não conectar** este projeto com Animerdash.
- Não tentar construir o jogo inteiro em um único ciclo.
- Entregar por incrementos pequenos e testáveis.

## Regras de Implementação (Unity 2D + C#)
- Usar scripts simples, limpos e modulares.
- Separar responsabilidades por script.
- Evitar scripts gigantes e acoplamento excessivo.
- Priorizar arquitetura que facilite manutenção e expansão.

## Regras de Conteúdo e Assets
- Usar placeholders simples quando não houver arte final.
- Não utilizar assets externos pagos.
- Não depender de pacotes externos sem necessidade clara.

## Regras de Processo
- Antes de implementar sistemas, manter documentação do contexto e escopo atualizada.
- Ao criar novas tarefas, sempre explicar:
  - Arquivos criados/alterados.
  - Como testar o que foi implementado.

## Princípios para Próximas Entregas
- Primeiro validar jogabilidade base do Bosque.
- Depois evoluir para conteúdo e refinamentos.
- Evitar otimizações prematuras.
